# 格ゲー認知タイプ診断 v2（SF6）

全43問の格闘ゲーム認知スタイル診断。MBTI16タイプ × 5層モデル（認知タイプ・勝利への美学・メンタル崩壊・成長曲線）を「素質の傾向／今の傾向」として分析します。

## 構成

- **Next.js 14（App Router）**
- `app/page.jsx` … 診断アプリ本体（全機能）
- `app/layout.jsx` … ルートレイアウト・メタデータ
- 依存：`react`, `react-dom`, `next`, `recharts`

## ローカルで動かす

```bash
npm install
npm run dev
```

http://localhost:3000 で表示。

## Vercel へのデプロイ

1. このフォルダを新規 GitHub リポジトリにプッシュ
2. Vercel で「New Project」→ そのリポジトリを選択
3. フレームワークは Next.js が自動検出される。そのままデプロイ
4. 発行された URL（例：sf6-diag-v2.vercel.app）で公開

既存の本番（60問版 / sf6-diag.vercel.app）はそのまま残し、新版が安定してから切り替える運用を想定。

## この最小構成（C）でできること

- 基本情報入力（メインキャラ30体・ランク帯）
- 全43問（MBTI16 + 実測：認知10 / 美学5 / メンタル5 / 成長7）
- 拮抗軸の追加質問（中立だらけの回答を救済）
- MBTI判定（左右回答数による公平なタイブレーク）
- 5層の理論導出 + 実測併記（素質の傾向／今の傾向）
- ズレ解釈・弱判定（実測が割れた時は素質を軸に表示）
- 第2部：キャラ × タイプの戦い方（30キャラ・核＋4認知相性）
- 結果ページでのキャラ切り替え（サブキャラも閲覧可）
- X 投稿 / 結果コピー

## 後から追加する項目（本番フェーズ）

- **集計データの永続化**：現在は集計DB未接続（管理者画面に注記表示）。Supabase 等に接続すると回答が集計される。`saveRecord` / `loadRecords`（page.jsx 内）を Supabase 呼び出しに置き換える
- **結果の保存・復元**：結果表示時に localStorage へ保存し、10分以内の再訪なら復元（離脱救済）
- **OGP画像**：SNS共有時のサムネイル
- **SITE_URL**：page.jsx 内の `SITE_URL` を本番URLに差し替え（X共有リンク用）

## 管理者集計画面へのアクセス

- タイトルを5回タップ（テスト用）
- または画面下部の入力欄に合言葉（page.jsx 内で設定。本番運用前に変更すること）
